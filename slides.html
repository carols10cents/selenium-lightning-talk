Why we started using Selenium

  • We make web applications that we wanted to test
  • No good way to test these besides in the browser (not rails)
  • Motto: We don't do manual testing!
  • When in Rails: to test JavaScript
  • Full stack integration testing
  • Fun to watch the green movie

Why you might not want to use Selenium

  • Opening and using a browser is slow compared to capybara+rack-test
  • Might be overkill, like view specs can be
  • Can be brittle
  • Still can't do subjective testing like "page looks pretty"

In Ruby: Tools

  • selenium-rc java tool + selenium-client gem
OR
  • selenium-webdriver gem (aka selenium 2.0, a merge with webdriver)
PLUS
  • Test harness of your choice (RSpec supports creating nice reports)

Ways to control the browser

  • .open
  • Locators of different types
  • .click
  • .type/.check/.select (form interactions)
  • .wait_for
  • ... and more, see Selenium::Client::GeneratedDriver and Selenium::Client::Idiomatic

Ways to test outcomes

  • get_text, is_text_present
  • is_element_present, get_xpath_count
  • is_visible
  • checked?, is_editable

Demo time!
